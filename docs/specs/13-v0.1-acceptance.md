# v0.1 Acceptance Specification

## 1. Open Existing Project acceptance

Using Praxis Studio repository itself:

```text
1. Repository scan succeeds.
2. FACT memory is written.
3. INFERENCE memory is written.
4. Architecture model candidate is generated.
5. Architecture views are projected from memory/model.
6. Project plan view is projected from specs/model gaps.
7. User can confirm memory records.
8. Confirmed records affect subsequent projections.
9. Explain mode can explain selected architecture node.
10. Plan mode can generate candidate plan.
11. Apply mode can write approved memory/spec/task changes only.
12. Coding task file can be generated.
```

## 2. Create New Project acceptance

```text
1. User enters product idea.
2. Idea memory is written.
3. Product model candidate is generated.
4. Domain model candidate is generated.
5. User confirms core distinctions.
6. Architecture model candidate is generated.
7. Specs are generated from confirmed memory/model.
8. Graph views are projected.
9. Project plan is generated.
10. Skeleton is created only after confirmation gates.
11. TASK-0001.md is generated.
```

## 3. Negative acceptance

Praxis v0.1 must not:

```text
- treat graph nodes as source of truth
- treat unconfirmed Agent output as confirmed memory
- generate code directly from raw idea
- modify existing source code automatically
- hide whether a relation is FACT or INFERENCE
- generate graph nodes without source memory IDs
```

# Praxis Studio Logo Candidate V5

Status: Superseded candidate.

Superseded by: `docs/brand/praxis-logo-candidate-v6.md`

Reason: user feedback found the memory-axis direction simple but too abstract. V6 keeps the order but restores readable project-memory meaning through a document object connected to governed memory nodes.

This candidate responds to the risk that V4 became simple but still bracket-like. V5 removes the enclosing cell and keeps only a governed memory axis.

## Concept

The mark is a minimal Memory Axis:

- The vertical amber axis represents Project Memory across its version timeline.
- The three cyan nodes represent FACT, CANDIDATE and CONFIRMED memory as ordered governance states.
- The horizontal amber rail represents controlled projection into graph, plan or task output.
- The endpoint keeps the mark from becoming a generic timeline while staying extremely simple.

## Design Rules

- One vertical axis.
- One projection rail.
- Three governance nodes.
- One output endpoint.
- No enclosing box.
- No document icon.
- No monogram target.

# Praxis Studio Logo Candidate V2

Status: Superseded candidate.

Superseded by: `docs/brand/praxis-logo-candidate-v3.md`

Reason: user feedback found the non-letter directions conceptually closer but still lacking visual order. V3 keeps the amber flow and cyan governance node semantics while shifting to a strict control grid.

This second logo direction responds to feedback that the first mark read too much like a `D` and was too constrained by a letterform goal. V2 keeps the accepted semantic direction of amber governed flow and cyan memory nodes, but removes the monogram requirement.

## Concept

The mark is a governed flow around a Project Memory core:

- The amber path represents Praxis' controlled development flow and studio craft.
- The cyan nodes represent FACT, CANDIDATE and CONFIRMED memory moving through governance.
- The central structured core represents durable Project Memory: normalized documents plus Git timeline, not a UI-only graph or `.distinction` cache.
- The outer path suggests graph projection without becoming a generic network diagram.

## Usage Notes

- Use the flow mark when Praxis needs a symbolic identity without leaning on a letter.
- Prefer the Memory Gate direction for the next refinement pass if the brand should emphasize docs-backed Project Memory as the product authority.
- Use the compact flow navigation mark at 24 to 32 pixels.
- Use the app icon candidate for desktop icon experiments only after confirmation.
- Keep this candidate as candidate memory until accepted by the project owner.

## Supersedes

This candidate supersedes the first `P`-like direction for brand exploration purposes, because the earlier mark carried too much `D`/letterform ambiguity.

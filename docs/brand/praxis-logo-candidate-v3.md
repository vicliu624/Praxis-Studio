# Praxis Studio Logo Candidate V3

Status: Superseded candidate.

Superseded by: `docs/brand/praxis-logo-candidate-v4.md`

Reason: user feedback found the ordered control-grid direction too complex. V4 keeps the order but simplifies the mark to one amber control cell and three cyan memory nodes.

This candidate responds to feedback that V2 still lacked order. V3 removes organic loops and scattered graph gestures. It uses a strict control grid: aligned nodes, orthogonal rails, and one central project memory block.

## Concept

The mark is a Project Memory control grid:

- The three cyan nodes form an ordered governance spine for FACT, CANDIDATE and CONFIRMED memory.
- The amber rails represent Praxis' controlled development flow and studio craft.
- The central block represents durable Project Memory: normalized documents plus Git timeline.
- The right amber boundary represents controlled output into graph projection, plan or task generation.

## Design Rules

- Orthogonal geometry only.
- Repeated spacing and aligned axes.
- One clear hierarchy: governance spine, memory core, output boundary.
- No monogram target.
- No organic orbit or scattered network shape.

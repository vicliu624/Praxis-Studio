C4 Code View 用来解释“架构组件如何落到少量关键代码锚点”。

Code View 必须从某一个 Component 下钻而来。它不是工程热点、不是代码浏览器、不是全量类图，也不是按关系数量排序的技术债榜。

## 必须解释

- 代码锚点是什么：文件、类、函数、组件、接口、schema、命令或配置。
- 这个锚点为什么对当前架构组件关键。
- 该锚点与上层 Component/Container 的关系。
- 变更该锚点可能影响的入口、协议、文档、配置、构建或测试。
- 哪些代码不在本视图范围内。

## 不要做

- 不要列出大量源码。
- 不要把 Code View 当作 Engineering Hotspot 或 Code Preview。
- 不要绕过软件结构模型对结构证据、运行链路和代码锚点的职责。
- 不要使用内部生成名、事实图 id、`C_xxx`、`n_xxx`、`fanIn`、`fanOut` 作为图中名称；图中名称必须优先使用真实代码名称、真实文件名或用户可理解的职责名。
- 不要让 Code View 脱离 Component 独立存在；如果无法说明它实现了哪个 Component，宁可不生成该 Code View。
